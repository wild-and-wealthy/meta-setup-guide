Meta Pixel + Conversion API Setup Guide

Install and verify your pixel, set up event tracking in GHL, and enable server-side CAPI — step by step.

Platform: GHL (GoHighLevel) only
Time required: 45–60 minutes
Device: Desktop recommended

Before you start — confirm you have all of this:

  • A personal Facebook account in good standing
  • A Meta Business Portfolio (Business Manager) set up
  • An Ad Account created inside Business Manager
  • Your GHL funnel live with real URLs (not draft/preview)
  • Admin access to your GHL account
  • A desktop or laptop — not a phone

01 /

What Is the Pixel — and Why It Matters

The Meta Pixel is a small piece of JavaScript code that you install on your funnel pages. Every time someone visits a page or takes an action (opts in, purchases), it fires a signal back to Meta.

Those signals teach Meta's algorithm who is engaging with your funnel — and over time, it uses that data to find more people like them. Without the pixel, your ads run blind. With it, your campaigns get smarter every day.

The further someone gets into your funnel, the more valuable the signal. Each event teaches Meta something different:

Base Code (PageView)
Goes on every page — fires when anyone visits. Your baseline signal.
Lead Event
Goes on your opt-in thank-you page — fires when someone submits their info. This is what feeds the learning phase.
InitiateCheckout Event
Goes on your sales/checkout page — fires when someone reaches the purchase step. Signals high intent.
Purchase Event
Goes on your order confirmation page — fires when a sale is completed.
The learning phase depends on this. Meta needs approximately 50 Lead events to exit the learning phase and stabilize your ad delivery. Every opt-in your pixel captures is feeding that count.

02 /

Create Your Pixel in Meta Events Manager

Step 1 — Open Events Manager

Go to business.facebook.com/events_manager and make sure you are logged into the right Facebook account (the one tied to your Business Portfolio).

Step 2 — Create a New Data Source
  1. Click + Connect Data Sources (left sidebar or green button)
  2. Select Web
  3. Click Connect
Step 3 — Name Your Pixel

Give your pixel a clear name — something like Your Name — WW Funnel Pixel. You can only have one pixel per ad account, so name it something you'll recognize.

Enter your website URL when prompted and click Continue.

Step 4 — Choose Your Setup Method

When Meta asks how you want to add the pixel to your website, select "Add code manually" — this gives you access to the actual code you'll need for GHL.

Step 5 — Copy Your Pixel ID

After your pixel is created, find your Pixel ID — it's a 15–16 digit number visible in Events Manager under your pixel name. You'll need this in the next section.

Already have a pixel? If you created one previously, do not create another. Go to Events Manager, find your existing pixel, and copy the Pixel ID from there. Using multiple pixels on the same funnel will cause tracking conflicts.

03 /

Install the Base Pixel Code in GHL

GHL Only These steps are specific to GoHighLevel. If you are not using GHL as your funnel platform, this guide does not apply.

The base pixel code needs to go into the header of every page in your funnel. In GHL, you do this at the funnel level — so it applies to all pages automatically.

Step 1 — Go to Your Funnel Settings
  1. In GHL, navigate to Funnels & Websites → Funnels
  2. Click on your funnel
  3. Go to the Settings tab (top right)
  4. Scroll to Header Tracking Code
Step 2 — Paste the Base Pixel Code

Copy the code below, replacing YOUR_PIXEL_ID with your actual 15–16 digit Pixel ID from Events Manager:

<!-- Meta Pixel Code -->
<script>
!function(f,b,e,v,n,t,s)
{if(f.fbq)return;n=f.fbq=function(){n.callMethod?
n.callMethod.apply(n,arguments):n.queue.push(arguments)};
if(!f._fbq)f._fbq=n;n.push=n;n.loaded=!0;n.version='2.0';
n.queue=[];t=b.createElement(e);t.async=!0;
t.src=v;s=b.getElementsByTagName(e)[0];
s.parentNode.insertBefore(t,s)}(window, document,'script',
'https://connect.facebook.net/en_US/fbevents.js');
fbq('init', 'YOUR_PIXEL_ID');
fbq('track', 'PageView');
</script>
<noscript><img height="1" width="1" style="display:none"
src="https://www.facebook.com/tr?id=YOUR_PIXEL_ID&ev=PageView&noscript=1"
/></noscript>
<!-- End Meta Pixel Code -->
Step 3 — Save

Click Save. The base pixel code is now installed across every page of your funnel.

Replace both instances of YOUR_PIXEL_ID. There are two places in the code above where your Pixel ID appears — the fbq('init', ...) line and the noscript image URL. Replace both.

04 /

Add Event Codes to Specific Pages

Event codes fire on specific pages to tell Meta when a meaningful action happened. You'll add these at the individual page level inside your funnel.

How to access a specific page's custom code in GHL
  1. Open your funnel → click into the page you want to edit
  2. Click Settings (gear icon) on the page editor
  3. Scroll to Custom Head Code or Custom JS/HTML
  4. Paste the event code below the base code (or on its own if base is at funnel level)
  5. Save and publish the page
Lead Event — Opt-In Thank You Page

Paste this on the page someone lands on immediately after they submit their opt-in form:

<script>
  fbq('track', 'Lead');
</script>
InitiateCheckout Event — Sales / Checkout Page

Paste this on the page where someone reaches the purchase step — before they've paid. This fires when someone shows high purchase intent:

<script>
  fbq('track', 'InitiateCheckout');
</script>
Purchase Event — Order Confirmation Page

Paste this on the page someone lands on after completing a purchase. Update the value field to match your product price:

<script>
  fbq('track', 'Purchase', {value: 0.00, currency: 'USD'});
</script>

05 /

Verify Your Pixel is Firing

Before moving on, confirm the pixel is actually working. The fastest way is the Meta Pixel Helper Chrome extension.

Step 1 — Install Meta Pixel Helper

In Chrome, search "Meta Pixel Helper extension" and install it from the Chrome Web Store. It's free and takes 30 seconds.

Step 2 — Visit Your Funnel Pages
  1. Open your funnel landing page in a new Chrome tab
  2. Click the Pixel Helper icon in your Chrome toolbar (top right)
  3. You should see a green checkmark with PageView — this confirms your base code is firing
  4. Navigate to your opt-in thank-you page
  5. Check Pixel Helper again — you should now see both PageView and Lead
  6. Navigate to your sales/checkout page — you should see InitiateCheckout firing
  7. Navigate to your order confirmation page — you should see Purchase firing
✅ What success looks like
Green icon with a count. Each event (PageView, Lead, InitiateCheckout, Purchase) appears with a green checkmark on the correct page. If PageView fires everywhere and each event fires only on its designated page, your pixel is installed correctly.
Seeing red or no events?
— Make sure your funnel is published (not in draft mode)
— Confirm you replaced YOUR_PIXEL_ID with your actual Pixel ID in both places
— Check that the code is in the Header section, not the Footer
— Try clearing your cache and reloading the page

06 /

Connect Your Pixel to Your Ad Account

Your pixel needs to be linked to your Ad Account so that Meta can use the data it collects for campaign targeting and optimization.

Step 1 — Go to Business Settings
  1. Go to business.facebook.com/settings
  2. In the left sidebar, scroll to Data Sources → click Pixels
  3. Select your pixel from the list
Step 2 — Add the Ad Account
  1. Click Add Assets
  2. Select Ad Accounts
  3. Check the box next to your Ad Account
  4. Click Add to confirm
Already connected? If you see your Ad Account already listed under the pixel's assets, you're good — nothing to change.

07 /

Set Up Conversion API (CAPI) in GHL

The Conversions API (CAPI) sends event data directly from your server to Meta — in addition to the browser pixel. This gives Meta a more complete, reliable signal, especially for users with ad blockers or iOS privacy restrictions.

GHL has native CAPI support built in. You do not need to write any code.

Step 1 — Connect Your Facebook Account in GHL
  1. In GHL, go to Settings → Integrations
  2. Find Facebook and click Connect
  3. Log into your personal Facebook account when prompted
  4. Grant the requested permissions and click Continue
Step 2 — Select Your Business Portfolio and Pixel
  1. After connecting, GHL will ask you to select your Business Manager
  2. Select the correct Business Portfolio (the one your Ad Account lives in)
  3. Select your Pixel from the dropdown
  4. Click Save
Step 3 — Enable Conversion API
  1. Still in Settings → Integrations → Facebook
  2. Look for the Conversions API toggle
  3. Turn it On
  4. Save your settings
Step 4 — Verify in Meta Events Manager
  1. Go back to Events Manager
  2. Select your pixel
  3. Under Overview, you should now see events coming in from two sources: Browser (your pixel) and Server (CAPI)
✅ What to look for
In Events Manager → your pixel → Overview, you'll see a "Connection Method" column. When CAPI is working, events show as both Browser and Server. This means Meta is getting double the signal — more reliable, more accurate.
Don't see the CAPI toggle in GHL? GHL UI updates frequently. If the toggle isn't where described, check Settings → Facebook → Advanced Settings, or search "Conversions API" in the GHL help center. The functionality exists — the navigation may have shifted.

Common Mistakes to Avoid

Using YOUR_PIXEL_ID literally

The most common issue. YOUR_PIXEL_ID in the code above is a placeholder — you must replace it with your actual 15–16 digit Pixel ID from Events Manager. The code will not work otherwise, and you won't get an error message — it will just silently fail.

Installing at the page level when you should use funnel-level

The base code should go in your funnel's header settings — this applies it to every page automatically. If you paste it only on one page, the other pages won't track. Event codes (Lead, Purchase) are the exception — those go on specific pages only.

Installing on a draft or preview URL

Your pixel only fires on the live, published version of your funnel. The preview URL and draft mode do not count. Always verify on the real, public URL.

Creating multiple pixels

One pixel per ad account. If you create a second pixel and install both on your funnel, you'll get duplicate events and skewed data. Check Events Manager before creating a new one — you may already have one from a previous setup.

Skipping the Pixel Helper verification step

Skipping verification means you might run ads for days before realizing the pixel isn't firing. Spend 5 minutes on this step. It saves hours of troubleshooting later.

Not enabling CAPI

Browser-only pixel tracking misses a growing percentage of users due to ad blockers, iOS 14+ privacy changes, and browser restrictions. CAPI fills those gaps. If you skipped Section 07, go back and do it — it takes 10 minutes and makes your data significantly more reliable.

Troubleshooting

What you see Most likely cause Fix
Pixel Helper shows no events Code not installed, or funnel is not published Confirm you saved the funnel header code in GHL and that your funnel is live — not in draft or preview mode
PageView fires but Lead does not Lead event code is missing or on the wrong page Check that the Lead event code is on the thank-you page that loads after someone submits the opt-in form — not the opt-in page itself
Pixel Helper shows red Pixel ID mismatch or code error Open the Pixel Helper details — it will show the specific error. Most often: wrong Pixel ID, or the code is malformed. Re-paste the code from scratch.
Events Manager shows 0 activity Pixel not connected to Ad Account, or no real page visits yet Confirm the pixel is connected to your Ad Account (Section 06), then visit your funnel pages yourself to trigger test events
CAPI toggle not visible in GHL GHL UI may have updated Search "Conversions API" in GHL's help center — the feature exists, the navigation path may have shifted. Bring your screenshot to Session 6 if you're stuck.
Events Manager shows duplicate events Two pixels installed on the same funnel Go to Events Manager and check how many pixels are associated with your account. Remove the duplicate, or remove one of the code installations from your funnel.
Still stuck after following these steps? Bring a screenshot of your Events Manager overview and the specific error to Session 6 (Buffer + Catch-Up Week). That's exactly what that session is for.
