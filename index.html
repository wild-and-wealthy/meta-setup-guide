What This Actually Does

Meta keeps your ad data locked behind an API — a secure door that requires a key before anything can read it. That key is called an access token.

Once you have your token and tell Claude where it lives, Claude can pull your spend, clicks, CTR, impressions, and more directly from Meta — no logging in, no screenshotting dashboards, no manual entry.

You generate the token once. It lasts about 60 days. Refreshing it when it expires takes 2 minutes.

02 /

Set Up Your Meta Developer Account

This is free and uses your existing Facebook login. You only do this once.

Step 1 — Go to Meta for Developers

Open this link in your browser:

→ developers.facebook.com

Click Get Started in the top right. Log in with the Facebook account that has access to the team's Meta ad account.

Step 2 — Complete the Developer Registration

Meta will ask you a few quick questions about how you plan to use the platform. Answer honestly — you're building tools to access your own ad data. Accept the terms and finish registration.

✓ Already done this? If you can see the Meta for Developers dashboard when you log in, skip ahead to Section 03.

03 /

Create Your Own Meta App

Meta requires an "app" to issue tokens. You'll create your own — it's free, takes 3 minutes, and never needs to be touched again. Each person on the team has their own app. This keeps your token yours and avoids any account entanglement.

Step 1 — Open the Graph API Explorer

This is the tool you'll use to create the app and generate your token. Open it here:

→ developers.facebook.com/tools/explorer/ Step 2 — Create a New App

At the top of the page, find the dropdown labeled Meta App. Click it and choose Create new app.

  1. When asked for a use case, select Other
  2. For app type, choose Business
  3. Name it something like YourName Meta KPI (e.g. Kim Meta KPI)
  4. Click through the remaining prompts — defaults are fine

Once created, select your new app from the Meta App dropdown at the top of the Explorer.

✓ You own this app. You're not joining anyone else's account. Your token is yours. If something breaks, you fix it independently — no waiting on anyone.

04 /

Generate Your Access Token

Step 1 — Add the Right Permission

In the Graph API Explorer, look for a section called Permissions or a button that says Add a Permission.

Search for and add: ads_management

This is the only permission you need. It gives read access to ad account data — nothing else.

Step 2 — Set Expiration to Never

Before generating, find the Token Expiration dropdown (it defaults to 60 days). Change it to Never.

This means you generate your token once and never have to repeat this process.

✓ Don't skip this step. The default is 60 days, which means your connection would silently break two months from now. Setting it to Never means it works until you intentionally revoke it.
Step 3 — Generate the Token

Click the blue Generate Access Token button.

A popup will appear asking you to log in and approve access. Click through everything — you're giving yourself permission to read your own ad data.

Your token will appear in the text box at the top of the page. It's a long string starting with EAG...

Step 4 — Copy and Save It

Copy the full token and paste it somewhere safe — a private note, your password manager, or anywhere only you can see it. You'll need it in the next step, and won't see it again after leaving this page.

⚠️ Keep this private Your token gives read access to the ad account. Don't paste it into any group chat, email, or shared doc. If you think it was ever exposed, generate a new one (Section 07) and update Doppler immediately.

05 /

Store It Securely in Doppler

You never paste your token directly into Claude's chat — that would leave it exposed in your conversation history. Instead, you store it in Doppler — a secrets manager that keeps credentials encrypted and out of your code. Claude reads from Doppler automatically when it runs any scripts.

Step 1 — Install Doppler

Open your Terminal and run:

brew install dopplerhq/cli/doppler

If you don't have Homebrew, install it first at brew.sh — one command, takes 2 minutes.

Step 2 — Create a Free Doppler Account

Go to doppler.com and sign up for free. Use the same email as your work accounts.

Step 3 — Log In from Terminal 
doppler login

This opens your browser and links your Terminal to your Doppler account.

Step 4 — Ask Natalie for the Project Invite

Natalie has a Doppler project set up for the team. She'll send you an invite link by email. Click it to join the project — you'll see it in your Doppler dashboard as wild-wealthy-kpi (or similar).

Step 5 — Connect the Project in Terminal

Navigate to your Claude working folder and run:

doppler setup

Choose the team project when prompted. This links your local environment to the right Doppler project.

Step 6 — Save Your Token 
doppler secrets set META_TOKEN

Doppler will prompt you to enter the value. Paste your token and press Enter. That's it — it's encrypted and stored. You never type it anywhere else.

Secret name
META_TOKEN
Value
Your EAG... token from Section 04
✓ How it works with Claude When you run Claude Code scripts, prefix the command with doppler run -- so secrets are injected automatically: 
doppler run -- claude
Claude can then read META_TOKEN as an environment variable — no hardcoding, no chat exposure.

06 /

Test the Connection

Let's confirm everything is working before you walk away.

Step 1 — Run the Test

In the Apps Script editor, find the function dropdown at the top of the page. Select testRun and click the ▶ Run button.

The first time you run anything, a permissions popup will appear. Click through:

  1. Review permissions
  2. Select your Google account
  3. Click Allow
Step 2 — Check the Execution Log

After it runs, look at the Execution Log panel at the bottom of the page. A successful connection looks like this:

Fetching Meta metrics for: 5/20/2026
Spend: $87.45 | Link Clicks: 312 | CTR: 4.2%
Sheet updated successfully.
✅ If you see that — you're connected.
Meta is talking to Claude. Your token works. You're done with setup.
Seeing a token error? Go back to Section 04 and make sure the full token was copied correctly — no spaces at the start or end, and nothing cut off.

07 /

If Your Token Ever Breaks

Since you set expiration to Never, your token won't expire on its own. The only reasons you'd need a new one: you accidentally shared it, Meta revoked it, or something got corrupted. Here's how to replace it in under 2 minutes.

  1. Go to developers.facebook.com/tools/explorer/
  2. Select your app from the dropdown
  3. Confirm ads_management is still checked
  4. Change expiration to Never again
  5. Click Generate Access Token — approve the popup
  6. Copy the new token
  7. In Terminal, run: doppler secrets set META_TOKEN
  8. Paste the new token when prompted, press Enter

Claude picks up the new token automatically on its next run. No other changes needed.

No 60-day reminders needed. If Claude stops returning Meta data and you're seeing a token error, this section is your fix. Otherwise, you should never need to come back here.

08 /

Troubleshooting

What you see What it means Fix
META_TOKEN not set or blank Doppler isn't injecting the secret — usually means doppler run -- wasn't used Make sure you're running Claude scripts as doppler run -- claude, and that doppler setup has been run in your working folder
Meta API error: invalid token Token was revoked, shared, or copied with an error Generate a new one following Section 07 — make sure expiration is set to Never — then update Doppler: doppler secrets set META_TOKEN
No permissions available The ads_management permission wasn't added to your app Go back to the Graph API Explorer, add the permission, regenerate the token
Can't find your app in the dropdown App wasn't created under the right Facebook account Make sure you're logged into Facebook as the account with ad account access, then create the app again
No Meta data returned No ad spend that day — normal if campaigns were paused Check Ads Manager to confirm. Not an error.
doppler: command not found Doppler CLI isn't installed Run brew install dopplerhq/cli/doppler and then doppler login
Still stuck? Drop a message in the team channel with a screenshot of the error from the Execution Log. Don't include your token — just the error text.

Auto-Fill Your KPI Google Sheet

Now that Claude can talk to Meta, you can take it one step further: set up a daily automation that pulls yesterday's ad data and writes it directly to the team's KPI sheet — every morning, automatically, before you open your laptop.

Step 1 — Paste the Automation Script

Ask Natalie for the meta_kpi_updater.gs script file. In your Google Sheet:

  1. Extensions → Apps Script
  2. Select all existing code and delete it
  3. Paste the script Natalie gave you
  4. Save (Cmd+S / Ctrl+S)
Step 2 — Run the Daily Trigger Setup

In the function dropdown, select setupDailyTrigger and click ▶ Run.

That's it. Every morning at 6am ET, the script wakes up, pulls the previous day's Meta data, and writes it to the matching date row in your sheet.

What gets written automatically every morning
Campaign Spend, Link Clicks, CTR, Impressions, CPM, Clicks (All), CTR (All), CTR (Link), Unique Clicks — all from Meta, all without you touching a thing.

The only columns you fill in manually are the ones that come from your own knowledge: Booked Calls, Attended, Closed, and Commission.
